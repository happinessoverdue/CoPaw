项目概述

CoPaw 是阿里巴巴通义实验室 AgentScope 团队开发的个人智能体助手，类似于 OpenClaw。它是一个可以本地或云端部署的 AI 助手，支持多通道对话（钉钉、飞书、QQ、Discord、iMessage 等），具备定时执行、技能扩展和记忆管理能力。

官网: https://copaw.agentscope.io

核心特性:

- 多通道对话 — 通过钉钉、飞书、QQ、Discord、iMessage 等与你对话
- 定时执行 — 按配置自动运行任务（心跳和 Cron 任务）
- 能力由 Skills 决定，有无限可能 — 内置定时任务、PDF 与表单、Word/Excel/PPT 文档处理、新闻摘要、文件阅读等，还可在 Skills Hub 中自定义扩展
- 数据全在本地 — 不依赖第三方托管
  
技术栈: Python 3.10+ (< 3.14), FastAPI, AgentScope, React (Console)

构建基础:

- AgentScope - 多智能体框架
- AgentScope Runtime - 运行时引擎
- ReMe - 记忆管理系统

智能体机制

CoPawAgent 架构

CoPawAgent 继承自 AgentScope 的 ReActAgent，是一个 ReAct (Reasoning + Acting) 模式的智能体：

sequenceDiagram
    participant U as 用户
    participant A as CoPawAgent
    participant T as Toolkit
    participant M as LLM

    U->>A: 发送消息
    A->>A: 处理文件/媒体块
    A->>A: 检查系统命令

    alt 是系统命令
        A->>A: CommandHandler处理
        A-->>U: 返回命令结果
    else 普通消息
        loop ReAct循环 (最多max_iters次)
            A->>M: 发送推理请求
            M-->>A: 返回响应/工具调用
            alt 需要调用工具
                A->>T: 执行工具
                T-->>A: 返回结果
            end
        end
        A-->>U: 返回最终回复
    end

记忆压缩

当上下文长度超过阈值（默认 70% 的 max_input_length）时，自动触发记忆压缩：

- 保留最近 N 条消息（默认 3）
- 将历史消息通过 LLM 生成摘要
- 用摘要替换历史消息

Cron 任务类型

- text — 到点向频道发一段固定文案
- agent — 到点向 CoPaw 提问，把回复发到频道
  
# 创建 text 任务
copaw cron create \
  --type text \
  --name "每日早安" \
  --cron "0 9 * * *" \
  --channel dingtalk \
  --target-user "用户ID" \
  --target-session "会话ID" \
  --text "早上好！"

# 创建 agent 任务
copaw cron create \
  --type agent \
  --name "检查待办" \
  --cron "0 */2 * * *" \
  --channel dingtalk \
  --target-user "用户ID" \
  --target-session "会话ID" \
  --text "我有什么待办事项？"

开发注意事项

添加新的频道

1. 在 src/copaw/app/channels/ 下创建新目录
2. 继承 BaseChannel 类
3. 实现必需的方法：
- build_agent_request_from_native() — 将原生消息转为 AgentRequest
- from_env() / from_config() — 从环境或配置创建实例
- start() / stop() — 生命周期
- send() — 发送消息
4. 在 registry.py 中注册: register_channel("my_channel", MyChannel)
  
添加新的技能

1. 在 src/copaw/agents/skills/ 下创建目录
2. 创建 SKILL.md 文件（必须包含 YAML front matter: name, description）
3. 可选添加 references/ 和 scripts/ 目录
4. 运行 copaw skills enable <name> 启用
  
添加新的工具

1. 在 src/copaw/agents/tools/ 下创建工具函数
2. 使用 @tool 装饰器或符合 AgentScope 工具规范
3. 在 src/copaw/agents/tools/__init__.py 中导出
4. 在 CoPawAgent._create_toolkit() 中注册

目标场景：电力调度自动化系统

- 与互联网完全隔离的内网环境

5. 智能体工作机制优化

为 CoPaw 的智能体增加复杂任务的**规划-执行-更新-执行-...**的机制。

当前机制：ReAct 模式

┌─────────────────────────────────────────────────────────┐
│                    ReAct 循环                            │
│  ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐              │
│  │思考 │ →  │行动 │ →  │观察 │ →  │思考 │ → ... → 输出 │
│  └─────┘    └─────┘    └─────┘    └─────┘              │
│     ↓          ↓                                      │
│  LLM推理    工具调用 (最多50轮)                          │
└─────────────────────────────────────────────────────────┘

特点：

- 反应式决策：每一步都是即时决策，没有预先规划
- 单轮循环：思考→行动→观察 不断循环直到完成任务
- 最大迭代次数：默认 50 次 (max_iters=50)
  
期望机制：规划-执行模式

┌─────────────────────────────────────────────────────────┐
│                 规划-执行-更新 循环                       │
│                                                         │
│  ┌──────┐    ┌──────────┐    ┌──────┐                  │
│  │ 规划 │ →  │ 执行子任务 │ →  │ 更新 │ → (循环)        │
│  └──────┘    └──────────┘    └──────┘                  │
│     ↓              ↓             ↓                     │
│  分解子任务     逐个执行       根据结果调整计划           │
└─────────────────────────────────────────────────────────┘

两种模式对比：
维度
ReAct (当前)
规划-执行-更新 (期望)
决策时机
每一步即时决策
先规划，再执行
任务分解
隐式、逐步分解
显式、一次性分解
计划调整
无明确计划
根据执行结果动态更新
复杂任务
可能迷失方向
有清晰的执行路径
可追溯性
难以追踪进度
可以追踪子任务完成状态

电力调度场景需要规划-执行模式的原因：

1. 操作规程严格：调度操作需要按固定流程执行
2. 多步骤任务：如"故障处置"需要 诊断→隔离→转供→恢复→记录
3. 可审计性：需要记录完整的操作计划
4. 中途可干预：调度员可能需要审核或调整计划


6. 人机交互
默认的控制台界面（http://127.0.0.1:8088/）偏向后台管理和简单对话使用，无法作为业务级界面使用。
需要结合我们系统的新人机和网页功能重新设计人机交互机制，可能的技术路线包括：
- 前端架构：使用 Vue 构建主界面，通过 WebSocket 与 CoPaw Backend 保持长连接，实现实时数据推送。
- Agent 技能封装：
  - 将“查询运行数据”、“业务分析问答”、“调用潮流计算”等封装为 CoPaw 的 Tools。
  - 将“调度知识库”、“应急预案”作为 RAG (检索增强生成) 的知识库挂载给 Agent。
- 人机回路：对于任何写操作（遥控、改定值），UI 层必须设计明确的“确认/取消”交互组件，严禁 Agent 全自动执行高危操作。

7. 典型应用场景
基于 CoPaw（智能体框架）开发电网调度系统的前端人机交互功能，需要结合传统电网调度系统的核心业务需求与AI 智能体的交互特性。根据2025-2026年新型电力系统的发展趋势（高比例新能源接入、源网荷储协同、AI深度赋能），以下是一些典型的功能模块：
1）全景可视化监控驾驶舱
这是调度员的“眼睛”，利用 CoPaw 的图表渲染能力或集成 ECharts/D3.js 等库。
- 电网拓扑动态展示：实时显示变电站、线路、开关状态（合/分），支持电压等级分层着色（如500kV红色，220kV橙色）。
- 关键运行指标看板：实时刷新频率、电压合格率、负荷率、新能源消纳率、备用容量等KPI。
- 潮流动态模拟：以动画形式展示电力潮流方向和大小，直观反映拥堵或重载线路。
- 地理信息映射 (GIS)：结合地图展示设备分布、气象灾害影响范围（台风、覆冰）、山火监测点。
2） 智能对话与指令交互
利用 CoPaw 的核心优势，将传统的“菜单点击”转变为“自然语言交互”。
- 自然语言查询：
  - “查询当前220kV XX变的主变负载率。”
  - “列出过去1小时频率越限的所有事件。”
- 意图识别与操作确认：
  - 用户：“拉开500kV XX线5011开关。”
  - CoPaw Agent：解析意图 -> 校验安规 -> 弹出二次确认卡片（显示风险等级、影响范围） -> 用户确认 -> 执行遥控。
- 多轮对话上下文：支持追问，例如先问“XX线路负荷多少？”，接着问“那它的N-1通过率呢？”，Agent需保持上下文记忆。
3） AI 辅助决策与预警
利用大模型和算法模型，从“被动监视”转向“主动建议”。
- 异常检测与根因分析：
  - 当告警发生时，Agent自动聚合相关告警，抑制风暴，并给出可能的原因（如：“疑似XX区雷雨导致线路跳闸，伴随3条重合闸失败”）。
- 负荷与新能源预测：
  - 展示未来24小时/7天的负荷曲线、风电/光伏出力预测，并标注置信区间。
- 调度策略推荐：
  - 在电力缺口或阻塞场景下，Agent生成多套调整方案（如：切负荷方案、启停机组方案、储能充放电方案），并对比各方案的经济性和安全性，供调度员选择。
- 事故推演与反事故演习：
  - 基于当前状态，模拟“如果XX线路断开，系统会怎样”，提前预知风险。
4） 操作控制与操作票管理
确保操作的安全性和规范性。
- 智能开票与校核：
  - 用户输入操作任务，Agent自动生成标准操作票步骤。
  - 自动进行防误校核（五防逻辑），拦截违规操作。
- 程序化操作进度追踪：
  - 可视化展示倒闸操作的执行进度，每一步骤的状态（未开始、执行中、成功、失败）。
- 紧急控制一键处置：
  - 针对低频低压减载、安稳装置动作等紧急情况，提供经过严格验证的“一键式”预案执行入口。
5） 多维报表与历史回溯
- 智能报表生成：
  - 通过对话生成日报、周报、月报，支持自定义维度（如“生成上周新能源弃风弃光分析报表”）。
- 事故追忆 (PDR)：
  - 支持时间轴拖拽，回放事故发生前后几分钟内的电网状态变化、告警序列和操作记录，用于事后分析。
- 趋势分析：
  - 交互式图表，支持缩放、对比不同时间段的数据。
6） 协同与通知
- 多渠道告警推送：
  - 严重故障时，除了界面弹窗，还可通过短信、App推送、语音电话通知相关人员。
- 交接班助手：
  - 自动生成交接班摘要，列出当前遗留问题、重要检修工作和注意事项。

- Reviewer (审核智能体)：结果审核与计划调整建议
